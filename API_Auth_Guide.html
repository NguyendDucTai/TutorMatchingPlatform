Hướng dẫn Tích hợp API Xác thực (Auth) cho Frontend

Tài liệu này dành cho đội ngũ Frontend (React, Vue, hoặc Vanilla JS) để biết cách kết nối giao diện với các API Xác thực (Authentication) của Backend.

Tất cả các API này đều gọi đến nhánh /api/auth/ và hỗ trợ định dạng dữ liệu application/json.

💡 Lưu ý: Mặc định Backend đang chạy ở https://localhost:5000 (hoặc cổng cấu hình tương tự). Hãy đảm bảo biến môi trường API_BASE_URL của Frontend trỏ đúng vào địa chỉ này.

1. Đăng ký tài khoản (Register)

Dùng cho màn hình Sign Up, cho phép người dùng đăng ký làm Student (Học viên) hoặc Tutor (Gia sư).

  • Endpoint: POST /api/auth/register
  • Quyền: Mọi người (AllowAnonymous)
  • Body (JSON):
{
  "fullName": "Nguyen Van A",
  "email": "nguyenvana@gmail.com",
  "password": "Password123!",
  "confirmPassword": "Password123!",
  "role": 1 
}
⚠️ Quan trọng - Quy ước thuộc tính Role (Quyền):
- 1 = Student (Học viên)
- 2 = Tutor (Gia sư)
(Lưu ý: API không cho phép đăng ký quyền Administrator - role 3. Mọi tài khoản Admin phải được cấp thủ công trong Database).

Response: Trả về một AuthenticationResult chứa thông tin user và mã token đăng nhập. Frontend có thể tự động cho user đăng nhập luôn sau khi đăng ký thành công mà không cần bắt họ gõ lại mật khẩu.

2. Đăng nhập (Login)

Dùng cho màn hình Sign In.

  • Endpoint: POST /api/auth/login
  • Quyền: Mọi người (AllowAnonymous)
  • Body (JSON):
{
  "email": "nguyenvana@gmail.com",
  "password": "Password123!"
}

Response Thành công (200 OK):

{
  "token": "eyJhbGciOiJIUzI1NiIsInR...", // Chuỗi JWT Token dùng để gọi các API khác
  "refreshToken": "xxxxx", // Dùng để gia hạn token nếu token chính hết hạn
  "success": true,
  "errors": [],
  "role": "Student", // Hoặc "Tutor", "Administrator"
  "userId": 10
}
💡 Mẹo: Frontend nên lưu chuỗi token này vào localStorage hoặc sessionStorage để dễ dàng lấy ra gắn vào Header (Authorization: Bearer <token>) cho các lần gọi API sau.

3. Quên mật khẩu (Forgot Password)

Dùng khi người dùng bấm vào nút "Quên mật khẩu", yêu cầu họ nhập Email để nhận mã khôi phục.

  • Endpoint: POST /api/auth/forgot-password
  • Quyền: Mọi người (AllowAnonymous)
  • Body (JSON):
{
  "email": "nguyenvana@gmail.com"
}

Response Thành công (200 OK):

{
  "success": true,
  "message": "Password reset token generated.",
  "resetToken": "A1B2C3D4..." // Trong môi trường DEV, Backend sẽ trả mã token trực tiếp về đây. Khi lên Production mã này sẽ gửi qua Email.
}

4. Đặt lại mật khẩu (Reset Password)

Sau khi có được mã resetToken (từ API Forgot Password hoặc từ Email), Frontend cung cấp 2 ô nhập mật khẩu mới để đổi mật khẩu.

  • Endpoint: POST /api/auth/reset-password
  • Quyền: Mọi người (AllowAnonymous)
  • Body (JSON):
{
  "token": "A1B2C3D4...", // Chuỗi token lấy từ bước Quên mật khẩu
  "newPassword": "NewPassword123!",
  "confirmPassword": "NewPassword123!"
}

Response Thành công (200 OK):

{
  "success": true,
  "message": "Password has been successfully reset."
}
🚨 Cảnh báo: Mật khẩu phải khớp nhau ở cả 2 trường newPassword và confirmPassword. Nếu gửi token sai hoặc hết hạn, API sẽ trả về lỗi 400 BadRequest.
